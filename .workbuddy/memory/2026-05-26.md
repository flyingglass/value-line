# 2026-05-26 Value Line 项目工作日志

## 数据架构重构 (18:50-)

### 问题
1. 中报数据硬编码在 engine.py → 无法支持多股票
2. AKShare `stock_financial_hk_report_em` 用 `indicator="年度"` 只拉年报
3. 营收结构硬编码在 generate_report.py

### 方案
**所有数据 → SQLite，engine.py 零硬编码**

### 改动

#### fetcher.py
- `fetch_hk_financials()`: `indicator="年度"` → `"全部"`  
  结果: income 表从 9年报 → 9年报+7中报 (2019-2025)
- SQLite 新增 `revenue_structure` 表 (code,year,dim_type,dim_name,amount,pct)
- 新增 `item_code` 列到 income/balance/cashflow 表 (通过 STD_ITEM_CODE 避免编码问题)
- 新增 `_supplement_dividend()` 补充港股股息缺口

#### engine.py
- 移除所有硬编码: `SEMI_ANNUAL_DATA`, `ANNUAL_PDF_DATA`, `DIV_MANUAL`
- `build_semi_annual()`: 从 income 表的 item_code 读取 (004001001=营收, 004025002=归母净利, 004027002=EPS)
- 营收结构从 revenue_structure 表读取
- 交叉校验改为: AKShare annual H1+H2 = Annual 的一致性检查
- 支持 code 参数切换股票

#### generate_report.py
- 营收结构改为从 DATA.revenue_structure 动态读取

#### insert_revenue.py (新增)
- 从PDF验证过的营收结构数据写入 SQLite

### 数据校验结果
- 7/7 H1+H2=Annual 通过 (0差异)
- 5个数据源: annual indicators + 半年度 income + 股息 + 营收结构 + HSI

### 数据流
```
fetcher.py → SQLite ← insert_revenue.py (PDF数据)
     ↓
engine.py → report_data.json
     ↓
generate_report.py → report.html (自包含)
```
